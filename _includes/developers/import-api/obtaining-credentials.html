This means that data successfully pushed using a given Import API access token will only ever be loaded into the schema or dataset created for that {{ connection-name }}. For example: You create an Import API {{ connection-name }} named `Customer Records`, which has a corresponding destination schema named `customer_records`. Any [data pushed to the Import API]({{ link.import-api.api | prepend: site.baseurl | append: site.data.import-api.core-objects.batch.anchor }}) made using the access token associated with the `Customer Records` {{ connection-name }} will only affect the data in the `customer_records` schema. Each Import API {{ connection-name }} is allowed a maximum of two active API access tokens at a time. This ensures that you can rotate your Import API access tokens when needed without interrupting replication.
